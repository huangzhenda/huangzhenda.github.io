  • 通过brew安装的话， 卸载相对简单。 通过 brew 命令行进行卸载。
  • 通过安装包安装的话，要删除相对应的文件。

大体上有几个路径： /usr/local/ , ~/Library/, /Library/, */private/var/db/receiptes
*

  • usr: usr 不是 user 的意思，而是 Unix Software Resource 的缩写，也就是 Unix 操作系统软件资源所放置的目录； 所有系统默认的软件都放在 /usr, 用户自行安装的 ，一般安装到/usr/local , 这样就容易管理。 因此 MySql 的安装目录就在 /usr/local/ 下。 所以需要删除这目录下，mysql相关的文件。
  • Library : MacOS专有的目录， 存放系统资源库以及用户的资源库相关的文件。/Library/StartupItems 定义了系统启动过程中运行的程序，/Library/LaunchAgents 定义了登录启动项，/Library/PreferencePanes 定义了系统偏好设置面板中的配置项，/Library/Receipts 存储的是应用的”票据“，类似于权限配置文件，以此类推 /private/var/db/receipts/ 里存放的也是类似的权限配置文件。

java学习笔记-基础

标识符与关键字

  • 标识符定义：
    • 字母、数字、_、$
    • 以英文字母开头。
  • 关键字： 系统对一些结构的描述处理，有特殊含义。

数据类型

  • 基础类型
    • 整型： byte 、 short 、 int 、long -> 默认值 0
    • 浮点型： float 、double -> 默认值：0.0
    • 布尔型： boolean -> 默认值： false
    • 字符型： char -> 默认值： \u0000
  • 引用类型
    • 数组、类型、接口 -> 默认值： null

数据范围！！！！

  • 如何处理超出的数据范围？ -> 如果超过最大的保存范围，会进行循环。 在java中成为数据溢出。
  • 解决数据溢出：
    • 在操作的时候预估数据范围，如果发现范围不够就使用更大的范围。
    • 可以直接追击 L 或者 long的转型。
  • 所有的数据类型进行自动转型的时候都是由小类型向大类型进行自动转换处理的。
  • 浮点型，默认是double。float需要添加 f/F。 float不精确。

运算

  • 数字运算
  • 条件运算
  • 逻辑运算
  • 位运算 ： 可以直接进行二进制数据的计算处理。
    • 与&、或｜、异或^、反码～、移位>> <<

逻辑控制

  • 条件 if
  • switch开关
  • 循环
    • while
    • do …… while
    • for

Java学习笔记-JShell

介绍

JShell 是java提供的 交互式编程工具。 官方解释：

The Java Shell tool (JShell) is an interactive tool for learning the Java programming language and prototyping Java code. JShell is a Read-Evaluate-Print Loop (REPL), which evaluates declarations, statements, and expressions as they are entered and immediately shows the results. The tool is run from the command line.

有了这种交互式编程工具，我们无需创建很多结构化的代码。我们只需要编写核心的代码，就能够很快的得到我们要的结果，验证我们的代码。

Java学习笔记 - 概述

概念

官方网址

  • [https://www.java.com] (https://www.java.com)
  • [https://www.oracle.com/java/] (https://www.oracle.com/java/)

Java SE/ Java EE/ Java ME

SUN公司给了三个不同版本。

  • Java SE: Standard Edition
  • Java EE: Enterprise Edition
  • Java ME: Micro Edition

三者的关系：

  • Java EE
    • Java SE
      • Java ME

Java SE就是标准版本， 包含标准的 JVM 和 标准库， 而 Java EE是企业版本， 它在Java SE的基础上，加了大量的API和库，以便方便开发 Web应用 、数据库、消息服务 等。 Java ME是针对嵌入式设备的 “瘦身版本”。 Java SE的标准库无法在Java ME上使用， Java ME的虚拟机也是“瘦身版”。

JDK & JRE

  • JDK: Java Development Kit
  • JRE: java Runtime Enviroment

JRE: JVM + Runtime Library。 JVM就是虚拟机，用来运行Java字节码的。
JDK： 包含了JRE, 另外还提供了编译器， 调试器等开发工具。 编译器能将Java源码编译成字节码。

JSR & JCP

JSR规范： Java Specification Request
JCP组织： Java Community Process

为了保证Java语言的规范性，SUN公司搞了一个JSR规范，凡事想给Java平台加一一个功能， 比如数据库的功能， 大家要先创建一个JSR规范， 定义好接口， 这样， 各个数据库厂商都按照规范写Java驱动程序，开发者就不用担心自己写的数据库代码在MySQL能跑， 却不能跑在PostgreSQL上。

JCP组织，就是来审核JSR的组织。

RI && TCK

  • RI: Reference Implementation
  • TCK: Technology Compatibility Kit

Java的标准库都有哪些？

[Java 14] (https://docs.oracle.com/en/java/javase/14/docs/api/index.html)

Java EE

  • Java EE最核心的组件是基于 Servlet标准 的 Web服务器 ，开发者编写的应用程序是基于Servlet API并运行在Web服务器内部的。

  • 支持Servlet API的Web服务器。 常用的服务器有：

    • Tomcat: 由Apache开发的开源免费服务器
    • Jetty: 由Eclipse开发的开源免费服务器
    • GlassFish： 一个开源的全功能JavaEE服务器

如何免费的搭建个人的博客

技术的选择

查找了下，目前搭建博客，主要用到以下几种方式：

  1. WordPress : 是一个以PHP和MySQL为平台的自由开源的博客软件和内容管理系统
  2. Jekyll : 简单静态博客网站生成器， 是用ruby写的
  3. octopress
  4. hexo

WordPress 太重， octopress 已经不怎么维护了， Jekyll 用的是ruby写的， hexo 用的是js。 相对于ruby，js会更熟悉点。最终选择 hexo来搭建博客

基于hexo搭建的博客

hexo是一个快速，简洁且高效的博客框架， 用Node.js写的，支持Markdown，部署安装都很简单，且拥有丰富的插件.

官方地址：https://hexo.io/zh-cn/

需要用到的工具

  1. Git
  2. Node.js
  3. GitHub Pages

Hexo的安装，部署，发布博文等

具体操作可以直接到官网上的文档上查询，步骤都写的很详细。

多台设备同步

可以用百度云类似的云存储，自动备份更新。只要在其他机子上把环境弄好，然后就可以同步写作了。

NEXT主题
